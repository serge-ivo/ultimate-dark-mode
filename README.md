# Ultimate Dark Mode

A Chrome extension that brings dark mode to every website on the internet.

## Vision

Ultimate Dark Mode is a production-quality, AI-built, open-source dark mode extension for Chrome.

**What makes it different from Dark Reader:**

- **No blind color inversion.** Dark Reader applies a generic algorithm that often breaks sites. For example, Google Sheets already has a dark frame/header — Dark Reader inverts it back to bright, while only darkening the content area. This looks terrible. Our extension never blindly inverts. Every site gets an appropriate, tested treatment.
- **Per-domain toggle that actually works.** One click in the popup to enable/disable dark mode for the current domain. Don't like the result on one site? Disable it there without turning off the whole extension. Dark Reader buries this.
- **AI-generated, screenshot-validated overrides.** Each supported site gets a custom CSS override generated by AI, tested with automated screenshots, and validated before shipping. Not a one-size-fits-all algorithm.

The core UX principles:
1. **One click to toggle dark mode on any site, per domain.**
2. **Never make a site look worse than it did without the extension.**

## Core Features

- **Per-domain toggle** — the primary interaction: click the extension icon to turn dark mode on/off for the current domain. State is remembered per domain.
- **No blind inversion** — we never apply `filter: invert()` or similar blanket transforms. Every dark mode treatment is intentional.
- **Universal coverage** — works on any website, including complex web apps (Gmail, GitHub, Notion, Slack, etc.)
- **AI-curated site overrides** — popular sites get custom CSS overrides generated by AI from screenshot analysis, tested automatically, and validated before merge.
- **Three-layer dark mode engine:**
  1. **Layer 1: Native dark mode activation** — detects sites with built-in `prefers-color-scheme: dark` support and forces it on. Zero cost, perfect results. Respects the site's own dark theme instead of fighting it.
  2. **Layer 2: Modern CSS transformation** — for sites without native dark mode, uses `oklch()` relative color syntax, `color-mix()`, and `@layer` to intelligently remap colors. Not inversion — remapping light colors to dark equivalents while preserving hue and contrast relationships.
  3. **Layer 3: JS-assisted analysis** — fallback for complex sites. `MutationObserver` for dynamic content, Shadow DOM support via `adoptedStyleSheets`, image/media detection.
- **Performance-first** — CSS-first approach means minimal JS overhead, no layout shifts, instant activation
- **Open source & AI-operated** — built with AI, maintained by GitHub agents

## Why Not Just Invert?

Color inversion breaks real websites because it's context-blind:

| Scenario | Inversion result | Our approach |
|---|---|---|
| Site already has dark header | Header becomes bright again | Detect existing dark elements, leave them alone |
| Light content area with data | Content darkens (good) but images invert | Remap background/text colors, preserve media |
| Partial dark mode (e.g. Google Sheets) | Some parts flip wrong way | Custom override that darkens only the light parts |
| Colored buttons/badges | Colors shift unpredictably | Preserve hue, adjust only lightness via oklch |
| Dark text on dark background (post-inversion) | Unreadable | Never happens — we control contrast ratios |

The key insight: **dark mode is not the inverse of light mode.** It requires understanding which elements are backgrounds, which are content, which are already dark, and which are media. A generic algorithm cannot do this reliably. Site-specific, AI-validated overrides can.

## Per-Domain Toggle — How It Works

```
User visits example.com
  → Extension checks storage: is dark mode enabled for example.com?
  → If enabled: inject dark mode styles
  → If disabled (or first visit): do nothing (default OFF)

User clicks extension icon
  → Popup shows toggle switch for current domain
  → Toggle ON: dark mode activates immediately, state saved
  → Toggle OFF: dark mode removed immediately, state saved

User visits example.com again later
  → Remembered: dark mode is ON → auto-activates
```

**Default behavior:** dark mode is OFF for all sites. Users opt-in per domain. This avoids breaking sites and gives users full control.

## Contributing

### GitHub Agents Pipeline

The override library grows through an automated pipeline:

```
1. User opens issue: "Dark mode broken on docs.google.com"
   OR: scheduled agent crawls top 1000 sites

2. GitHub agent spins up headless Chrome via Puppeteer
   → Takes screenshot of site (light mode baseline)
   → Analyzes DOM structure, existing colors, dark mode support

3. AI generates a site-specific CSS override
   → Understands which parts are already dark
   → Only darkens the light parts
   → Preserves images, media, brand colors

4. Agent applies override, takes "after" screenshot
   → Validates: no broken layouts, readable contrast, images preserved

5. Agent opens PR with:
   → The CSS override file
   → Before/after screenshot comparison
   → Automated visual regression test results

6. CI runs visual regression + accessibility checks
   → Auto-merge if all checks pass
   → Flag for human review if confidence is low
```
