The Problem

Why not just invert colors?

Color inversion is context-blind. Extensions like Dark Reader apply a generic algorithm that often breaks real websites — inverting already-dark headers back to bright, mangling images, and producing unreadable text. Dark mode is not the inverse of light mode.

Scenario Inversion result Our approach
Site already has dark header Header becomes bright again Detect existing dark elements, leave them alone
Light content area with data Content darkens but images invert Remap background/text colors, preserve media
Partial dark mode (e.g. Google Sheets) Some parts flip the wrong way Custom override darkens only the light parts
Colored buttons & badges Colors shift unpredictably Preserve hue, adjust only lightness via oklch
Dark text on dark bg (post-inversion) Unreadable Never happens — we control contrast ratios
Architecture

A three-layer dark mode engine

Instead of one blunt algorithm, Ultimate Dark Mode uses a layered approach — choosing the lightest-weight technique that works for each site.

1

Native Dark Mode Activation

Many sites already ship a prefers-color-scheme: dark stylesheet. Layer 1 detects this and forces it on — zero runtime cost, pixel-perfect results. Why fight the site when it already has a dark theme?

2

oklch CSS Color Remapping

For sites without native dark mode, we use modern CSS — oklch() relative color syntax, color-mix(), and @layer — to intelligently remap colors. This is not inversion: light colors become dark equivalents while preserving hue and contrast relationships.

3

JS-Assisted Fallback

Complex web apps with dynamic content, Shadow DOM, or inline styles get a lightweight JavaScript assist. MutationObserver handles dynamic content, adoptedStyleSheets reaches into Shadow DOM, and image detection ensures media is never touched.

Per-Domain Control

One click. Per site. Remembered forever.

Dark mode is off by default. Click the extension icon to enable it for the current domain. Your choice is saved and auto-applied on every future visit.

Don't like how dark mode looks on one particular site? Turn it off there without affecting the rest of the web. No digging through settings. No global on/off compromise.

AI-Powered

Community-driven override pipeline

When dark mode doesn't look right on a site, users report it. An AI agent takes it from there — analyzing screenshots, generating CSS, and opening a PR with before/after proof.

1

User reports a broken site

Opens a GitHub issue with a screenshot of the problem. "Dark mode looks wrong on docs.google.com."

2

AI agent analyzes the screenshot

Spins up headless Chrome via Puppeteer. Takes a baseline screenshot, analyzes the DOM structure, existing colors, and dark mode support.

3

Generates a site-specific CSS override

Understands which parts are already dark. Only darkens the light parts. Preserves images, media, and brand colors.

4

Opens a PR with before/after comparison

The PR includes the CSS override file, before/after screenshots, and automated visual regression test results.

5

User verifies, agent iterates

If the result isn't perfect, the user leaves feedback. The agent refines the override and updates the PR until it's right.

6

Auto-merge when confirmed

Once CI passes visual regression and accessibility checks, and the user approves, the override auto-merges and ships.

Get Started

Install in under a minute

Ultimate Dark Mode isn't on the Chrome Web Store yet. Install it as an unpacked extension in developer mode.

Step 1

Clone the repo

Run git clone from the GitHub repository or download the ZIP.

Step 2

Open Chrome extensions

Navigate to chrome://extensions/ and enable Developer mode in the top right.

Step 3

Load unpacked

Click "Load unpacked" and select the cloned directory.

Step 4

Start toggling

Click the extension icon on any site to enable dark mode for that domain. That's it.

🐞

Report a broken site

Found a site where dark mode doesn't look right? Open an issue with a screenshot. Our AI agent will generate a fix.

Under the Hood

Modern CSS, zero dependencies

Built on cutting-edge CSS features for perceptually accurate color transformation — no npm packages in production.

oklch()

Perceptually uniform color space for accurate lightness remapping without hue shifts.

color-mix()

Blend colors toward a dark palette natively in CSS, no JavaScript needed.

CSS @layer

Clean override cascade that works with site styles without abusing !important.

View Transitions

Smooth, animated toggle between light and dark modes with zero layout shift.

Manifest V3

Chrome's modern extension platform with service workers and declarative APIs.

adoptedStyleSheets

Reach into Shadow DOM to apply dark mode to web components and custom elements.

MutationObserver

Watch for dynamically added content and apply dark mode styles instantly.

chrome.storage.sync

Per-domain settings synced across all your Chrome devices automatically.
